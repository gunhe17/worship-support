reference/tailwind-example 디자인 힌트

사용법
- 아래는 각 HTML 샘플을 "어떤 디자인 상황"에서 참고하면 좋은지 빠르게 찾기 위한 메모입니다.
- 형식: 추천 상황 / 핵심 컴포넌트 / 특징

01.html
- 추천 상황: 서비스 랜딩 첫 화면(히어로) + 상단 내비게이션 + 모바일 메뉴가 한 번에 필요한 경우
- 핵심 컴포넌트: Hero Section, Top Navigation, Mobile Drawer(Dialog), CTA 버튼, 공지형 배지
- 특징: 다크 배경 + 블러 그라디언트, 메시지 중심의 강한 첫인상, 전형적인 SaaS 랜딩 시작점

02.html
- 추천 상황: 제품 핵심 기능을 "설명 텍스트 + 스크린샷" 2열로 소개할 때
- 핵심 컴포넌트: Feature Intro, 아이콘 리스트(dl), Product Screenshot
- 특징: 왼쪽 설명/오른쪽 이미지 구조가 안정적이라 기능 소개 섹션으로 바로 쓰기 좋음

03.html
- 추천 상황: 기능 4~6개를 동일한 중요도로 카드형 나열하고 싶을 때
- 핵심 컴포넌트: Feature Grid(2열), 아이콘 박스, 섹션 타이틀
- 특징: 균등 그리드 구조, 반복 가능한 패턴이라 확장/축소가 쉬움

04.html
- 추천 상황: 중간 구간에서 행동 유도(가입/시작)용 배너가 필요할 때
- 핵심 컴포넌트: CTA Banner, 보조 설명, 스크린샷 프리뷰
- 특징: 한 덩어리 프로모션 블록 형태, 페이지 중간 전환 포인트에 적합

05.html
- 추천 상황: 여러 장점을 시각적으로 강조하는 "벤토(Bento)" 레이아웃이 필요할 때
- 핵심 컴포넌트: Bento Grid, 기능 카드, 디바이스/코드 프리뷰 영역
- 특징: 카드 크기/비중 차이로 정보 우선순위를 표현하기 좋음

06.html
- 추천 상황: 요금제 비교(2~3 플랜) 섹션이 필요할 때
- 핵심 컴포넌트: Pricing Cards, 체크리스트, 대표 CTA 버튼
- 특징: 추천 플랜 강조가 쉬운 구조, 가격 페이지 기본 템플릿으로 적합

07.html
- 추천 상황: 채용/회사 소개 상단에 임팩트 있는 비주얼+지표를 넣고 싶을 때
- 핵심 컴포넌트: Background Hero, 링크 목록, KPI Stats(dl)
- 특징: 사진 오버레이 + 수치 강조로 브랜드 신뢰/규모감 전달에 유리

08.html
- 추천 상황: 뉴스레터/이메일 구독 유도 섹션이 필요할 때
- 핵심 컴포넌트: Email Input + Submit, 신뢰 포인트(아이콘 2개)
- 특징: 간단한 리드 수집 블록, 랜딩 하단 전환 영역으로 사용하기 좋음

09.html
- 추천 상황: 블로그/아티클 목록 프리뷰를 3열 카드로 보여줄 때
- 핵심 컴포넌트: Article Card, 카테고리 태그, 작성자 정보
- 특징: 콘텐츠 허브 첫 화면에 적합, 정보 밀도 대비 가독성이 좋음

10.html
- 추천 상황: 영업 문의/데모 신청 폼(긴 문의 폼)이 필요할 때
- 핵심 컴포넌트: Contact Form, 전화 국가코드 입력, 메시지 textarea, 약관 토글
- 특징: B2B 리드 폼에 가까운 구성, 필드가 많아도 레이아웃이 안정적

11.html
- 추천 상황: 팀/리더십 소개 섹션을 간결하게 보여줄 때
- 핵심 컴포넌트: People List, 아바타 + 이름 + 직책
- 특징: 프로필 카드가 가볍고 반복 배치가 쉬워 회사 소개 페이지에 적합

12.html
- 추천 상황: 상세 문서/가이드 페이지(텍스트 + 고정 이미지) 레이아웃이 필요할 때
- 핵심 컴포넌트: Long-form Content, Sticky Screenshot, 본문 리스트
- 특징: 블로그 본문/기능 설명 문서에 적합, 읽기 흐름이 길어도 안정적

13.html
- 추천 상황: 드롭다운/메가메뉴/모바일 아코디언이 있는 복합 헤더가 필요할 때
- 핵심 컴포넌트: Mega Menu(Popover), Mobile Dialog, Disclosure(아코디언)
- 특징: 네비게이션 요구사항이 복잡한 서비스형 사이트에 적합

14.html
- 추천 상황: 404/에러 페이지를 빠르게 구성할 때
- 핵심 컴포넌트: Error Hero, 기본 CTA(홈으로/문의)
- 특징: 가장 단순한 실패 상태 페이지 템플릿

15.html
- 추천 상황: 상세 페이지 상단 "타이틀 + 메타정보 + 액션 버튼" 헤더가 필요할 때
- 핵심 컴포넌트: Page Header, 메타 아이콘 리스트, 우측 액션 버튼군, 모바일 More 드롭다운
- 특징: 채용공고/문서/프로젝트 상세 상단에 재사용하기 좋음

16.html
- 추천 상황: 엔티티 상세 정보(라벨-값)를 정돈된 형태로 보여줄 때
- 핵심 컴포넌트: Description List(dl), 첨부파일 리스트
- 특징: 읽기 전용 상세 페이지(신청서/사용자 정보/설정 요약)에 적합

17.html
- 추천 상황: 사람/멤버 목록을 상태와 함께 보여줘야 할 때
- 핵심 컴포넌트: Stacked List, 아바타, 상태(Online/Last seen)
- 특징: 팀/참여자/담당자 목록 UI의 기본형으로 쓰기 좋음

18.html
- 추천 상황: 계정 설정처럼 섹션이 많은 복합 폼을 구성할 때
- 핵심 컴포넌트: Multi-section Form, 파일 업로드, 체크박스/라디오 그룹, 하단 액션
- 특징: 프로필/개인정보/알림설정을 한 페이지에서 다루는 표준 패턴

19.html
- 추천 상황: 사용자/담당자 선택처럼 아바타 포함 커스텀 셀렉트가 필요할 때
- 핵심 컴포넌트: Custom Select(el-select), 옵션 리스트, 선택 상태 체크 아이콘
- 특징: 기본 select보다 풍부한 표현이 가능, 할당/멤버 선택 UX에 적합

20.html
- 추천 상황: 로그인 페이지를 가장 빠르게 만들고 싶을 때
- 핵심 컴포넌트: Auth Form(Sign in), 이메일/비밀번호 입력, 보조 링크
- 특징: 단일 목적 페이지로 명확하고 구현 비용이 낮음

21.html
- 추천 상황: 상태/카테고리/태그 표시용 배지 스타일만 필요할 때
- 핵심 컴포넌트: Colored Badges
- 특징: 색상 팔레트 샘플 역할, 테이블/리스트 상태 라벨에 즉시 적용 가능

22.html
- 추천 상황: 기본 액션 드롭다운(설정, 지원, 로그아웃 등)이 필요할 때
- 핵심 컴포넌트: Dropdown Menu, 텍스트 액션 목록
- 특징: 가장 일반적인 1단 메뉴 패턴

23.html
- 추천 상황: 메뉴를 그룹(수정/이동/공유/삭제)으로 나눠야 할 때
- 핵심 컴포넌트: Grouped Dropdown, 섹션 구분선(divide-y)
- 특징: 액션 수가 많은 경우 인지 부하를 줄이기 좋음
